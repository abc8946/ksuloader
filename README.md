##   其次，**模块要现编现用，内核操作一动一查，因为你没解锁**
常见内核均已开启地址随机化，虽然的确有部分机型开了跟没开一样但你没解锁，你把上一次开机的硬编码地址加载进这一次的内核，*万一落到个写函数上你不炸了*
讲解一下各个脚本，其中两个是提取符号，用来收集未导出符号。原则上你可以用kallsyms完成所有链接工作，但gki内核各种幺蛾子，所以还是给他点工作吧（当然可能也没必要）
另一个脚本是把kallsyms的格式转化为.ld也就是链接器接受的格式，用来提前处理那些未导出的符号以免被系统拒绝加载
然后就是编译两个.c文件，各跑一边看看有没有激活，没激活可以试着把描述符发送给ksu管理器。看看它会不会识别到但我这边每次都激活了没法给各位测试
