# ksuloader
库内脚本可解决无法直接insmod含未导出符号的模块，两个c一个魔法值是从ksucall.rs上找到的，一个来自supercall.c所以都留着了，万一坂本不同两个都用
### 有坑，有坑，有坑！
故意留的，共2处ggou别来沾边
都是很明显的，一些顺序问题和一点常识，没看出来问题也不大，虽然把全零指针先写进去我也不知道会咋样
另外说明一下，装载.ko后可能管理器不识别，毕竟不是ksuinit挂上去的有些用户空间行为和修补boot不一致，所以才会用到那两个.c
ksu通过一个伪装的reboot调用来向用户空间暴露设备描述符，这里一开始装载完模块时或许因为管理器不知道？总之最后就是不激活，但dmesg已经有ksu相关日志且运行良好
两个.c都是通过人为调用特定参数的reboot让这个描述符留下痕迹，但据我所知描述符应该是进程私有我也没传给ksu管理器，所以最后还是没搞明白咋回事
